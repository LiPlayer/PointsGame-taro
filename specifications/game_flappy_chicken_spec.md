# 游戏开发规范：飞翔的小鸡 (Flappy Chicken)

本章节定义 **飞翔的小鸡** 的具体设计与参数。

## 1. 核心玩法逻辑 (Core Mechanics)

- **游戏模式**：**无尽积分挑战 (Infinite Score Attack)**
- **游戏目标**：控制小鸡穿越障碍,尽可能飞得更远
- **操作方式**：点击屏幕让小鸡向上飞,松开后自然下落
- **失败条件**：小鸡碰到障碍物或掉出屏幕
- **胜利条件**：无（游戏以失败结束结算分数）

## 2. 动态节奏控制 (The "Feel")

- **动态难度**：
  - 随着分数增加,障碍间距逐渐缩小
  - 障碍移动速度逐渐加快
  - 障碍高度差随机变化
- **操作反馈**：
  - 点击时小鸡有明显的上升动画
  - 下落时有自然的重力感
  - 穿越障碍时有轻微震动反馈

## 3. 视觉与UI设计 (Visuals & UI)

### 3.1 游戏元素
- **小鸡 (Chicken)**：
  - 可爱的卡通小鸡形象
  - 飞行时翅膀扇动动画
  - 契合"贵州炒鸡"品牌主题
- **障碍物 (Obstacles)**：
  - **悬浮云朵 (Floating Clouds)**：
    - **视觉表现**：**实心 (Solid)** 的 fluffy 云块。由多个圆形团聚而成，禁止出现镂空或缝隙。
    - **形状算法 (Gaussian Clustering)**：
        - 每个云朵包含约 15-20 个圆形 (Puffs)。
        - **团聚逻辑**：圆形位置遵循以云朵中心为基准的正态分布散布，确保云朵外轮廓圆润且无孤立小块。
        - **稳定性**：每个云朵使用固定种子 (Seed)，确保移动过程中形状不颤抖。
    - **分布算法 (Normal Distribution)**：
        - **位置 (Y)**：中心点位于屏幕 **上 1/4 (25%)** 或 **下 3/4 (75%)**。
        - **高度 (Height)**：均值为屏幕高度的 50%。
        - **宽度 (Width)**：均值 150px。
    - **玩法**：玩家需在云朵形成的"通道"中穿梭。
  - **独立闪电 (Independent Lightning)**：
    - **独立生成**：独立于云朵出现的障碍物，**可与云朵重叠**。
    - **分布算法**：遵循与云朵相同的 **正态分布 (Normal Distribution)** 逻辑（上 1/4 或 下 3/4 区域）。
    - **视觉**：柔和卡通风格 (Soft Cartoon)，白色核心+黄色光晕。
    - **时隐时现 (Blinking)**：按固定周期闪烁，只有 **可见 (High Visibility)** 时才具备碰撞体积。
- **背景 (Background)**：
  - 浅色系渐变色天空，**无地面 (No Ground)**。
  - **全屏适配**：逻辑分辨率 **360x800**，强制填满容器宽度与高度，不留黑边。
  - **移除逻辑**：云朵移除需包含安全边距（约 100px），确保所有散布的圆形完全离开屏幕后再销毁。

## 4. 胜负结算流程

- **Fail (失败)**：
  - **触发**：小鸡碰撞障碍或掉出屏幕
  - **视觉**：小鸡掉落动画
  - **流程**：短暂展示后跳转 Result 结算页,传递最终分数
